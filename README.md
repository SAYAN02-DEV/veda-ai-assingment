# VedaAI — AI-Powered Assignment Generator

VedaAI is a full-stack web application that lets teachers generate structured question papers using Google Gemini. Teachers configure subject, topic, difficulty, question types, and marks — the AI generates a formatted paper in the background, with real-time progress updates via WebSockets.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Next.js 15, Tailwind CSS, Framer Motion |
| Backend | Express.js (Node.js), TypeScript |
| AI | Google Gemini (via AI service) |
| Database | MongoDB (Mongoose) |
| Queue | BullMQ + Redis |
| Storage | Google Cloud Storage |
| PDF | Puppeteer + Handlebars |
| Real-time | Socket.IO |

## Local Development Setup

### Prerequisites

- [Node.js 22+](https://nodejs.org/)
- [Docker + Docker Compose](https://docs.docker.com/get-docker/)
- A `gcs-key.json` service account key file from Google Cloud (place it in the project root)

### 1. Clone the repository

```bash
git clone https://github.com/SimantiGhosh/veda-ai-assingment.git
cd veda-ai-assingment
```

### 2. Configure environment variables

Create a `.env` file in the project root:

```dotenv
NODE_ENV=development
PORT=4000

MONGODB_URI=<your-mongodb-uri>
REDIS_URL=<your-redis-url>
REDIS_PASSWORD=

JWT_SECRET=your_super_secret_key_here
JWT_EXPIRES_IN=7d

GCS_BUCKET_NAME=<your-gcs-bucket-name>
GCS_PROJECT_ID=<your-gcp-project-id>
GOOGLE_APPLICATION_CREDENTIALS=./gcs-key.json
GEMINI_API_KEY=<your-gemini-api-key>

GENERATION_QUEUE_CONCURRENCY=3
PDF_QUEUE_CONCURRENCY=5
```

Place your `gcs-key.json` file in the project root alongside `docker-compose.yml`.

### 3. Start the backend (Docker)

This starts the Express API, MongoDB, and Redis together:

```bash
docker compose up --build
```

The backend will be available at `http://localhost:4000`.

### 4. Start the frontend

In a separate terminal:

```bash
cd client
npm install
npm run dev
```

The frontend will be available at `http://localhost:3000`.

## How It Works

1. A teacher registers/logs in and creates an assignment by filling in subject, topic, marks, difficulty, and question types.
2. The backend queues a generation job via BullMQ.
3. A worker picks up the job, calls Gemini to generate a structured question paper, and saves it to MongoDB.
4. Real-time progress is pushed to the client via Socket.IO.
5. Once ready, the teacher can preview the paper and export it as a PDF (generated by Puppeteer).
6. PDFs are stored in Google Cloud Storage and a download link is returned.

## API Overview

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/v1/auth/register` | Register a new user |
| POST | `/api/v1/auth/login` | Login and receive JWT |
| POST | `/api/v1/assignments` | Create a new assignment |
| GET | `/api/v1/assignments` | List all assignments |
| GET | `/api/v1/assignments/:id/status` | Poll assignment status |
| GET | `/api/v1/assignments/:id/paper` | Get generated paper (JSON) |
| GET | `/api/v1/assignments/:id/paper/html` | Get generated paper (HTML) |
| POST | `/api/v1/assignments/:id/export-pdf` | Trigger PDF export |
| DELETE | `/api/v1/assignments/:id` | Delete an assignment |

All routes except auth require a `Bearer` token in the `Authorization` header.
